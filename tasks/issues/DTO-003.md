## :white_check_mark: Task Breakdown (실행 계획)
- [ ] **1. Zod 및 TypeScript 유틸 패키지 확인** — `npm install zod` (이미 설치 시 확인만)
- [ ] **2. 핵심 열거형(Enum-like) 스키마 정의** — `src/lib/schemas/recommendation.ts`:
  ```typescript
  import { z } from 'zod';

  // --- Enum-like 상수 ---
  export const directionSchema = z.enum(['BUY', 'SELL']);
  export type Direction = z.infer<typeof directionSchema>;

  export const confidenceScoreSchema = z.enum(['aggressive', 'balanced', 'conservative']);
  export type ConfidenceScore = z.infer<typeof confidenceScoreSchema>;

  export const cardStatusSchema = z.enum(['published', 'no_call', 'validation_failed']);
  export type CardStatus = z.infer<typeof cardStatusSchema>;
  ```

- [ ] **3. RecommendationCard 기본 필드 스키마 정의**
  ```typescript
  export const recommendationCardBaseSchema = z.object({
    id: z.string().cuid(),
    userId: z.string().cuid(),
    ticker: z.string().min(1).max(10),          // 종목 심볼 (예: AAPL)
    direction: directionSchema,                  // 'BUY' | 'SELL'

    // 진입가: 단일가 또는 범위 (상호 배타적, refine은 DTO-010)
    entryPrice: z.number().positive().nullable(), 
    entryRangeLow: z.number().positive().nullable(),
    entryRangeHigh: z.number().positive().nullable(),

    // 청산가: 단일가 또는 범위 (상호 배타적, refine은 DTO-010)
    targetPrice: z.number().positive().nullable(),
    targetRangeLow: z.number().positive().nullable(),
    targetRangeHigh: z.number().positive().nullable(),

    stopPrice: z.number().positive().nullable().optional(),

    holdDays: z.number().int().min(1).max(10),   // 보유 기간 1~10 정수
    confidenceScore: confidenceScoreSchema,       // 리스크 성향 단계
    reasonLine: z.string()
      .min(1, '한 줄 이유는 비어 있을 수 없습니다')
      .max(160, '한 줄 이유는 160자 이하이어야 합니다')
      .refine(
        (val) => val.trim().length > 0,
        '공백만으로 구성된 이유는 허용되지 않습니다'
      ),
    status: cardStatusSchema,
    createdAt: z.coerce.date(),
    validUntil: z.coerce.date(),
  });
  ```

## :gear: Technical & Non-Functional Constraints
- **Zod 버전:** `^3.22.0` 이상 (`.refine()`, `.coerce.date()` 지원 필수)
- **ENUM 대체:** SRS §6.2 규칙에 따라 Prisma String 필드를 Zod `z.enum()`으로 코드 레벨 검증
- **가격 택 1 규칙:** 본 태스크에서는 개별 필드의 `positive().nullable()` 만 적용. **상호 배타 규칙(`entryPrice XOR (entryRangeLow AND entryRangeHigh)`)은 DTO-010에서 `.refine()`으로 구현**
- **reasonLine 공백 검증:** `.min(1)`만으로는 공백 문자열을 차단할 수 없으므로 `.refine(val => val.trim().length > 0)` 추가 필수
- **TypeScript 타입 추출:** `z.infer<typeof schema>`를 통해 런타임 검증과 컴파일 타임 타입 안전성을 동시에 확보
- **재사용성:** `directionSchema`, `confidenceScoreSchema`는 다른 DTO(DTO-002, DTO-004)에서도 import하여 사용
